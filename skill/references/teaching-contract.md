# Teaching Contract

这是 `learning-project-tutor` 的最小教学契约。

## What this skill does
- 把 source 拆成小步 lesson
- 每节课都要求正式 response
- 先做 diagnosis，再决定 `advance` / `bridge` / `reframe`
- topic 结束后必须进入 synthesis / transfer / articulation / closure

## Non-negotiables
1. 先读 `topic_state.yaml`
2. 用状态驱动动作，而不是用“文件存在”驱动动作
3. 没有 formal response，不进入 diagnosis
4. 没有 diagnosis，不进入下一课
5. QA 只能辅助解释 response，不能代替 response
6. lesson 只覆盖一个小认知单元
7. 问题优先诊断理解，不优先考记忆

## Lesson design
- 先解释为什么，再解释是什么
- 控制容量，避免一课塞多个并行主题
- 术语只给最小必要解释
- 问题至少覆盖复述、因果、区分、迁移、边界中的三个维度

## Diagnosis design
每次 diagnosis 至少判断：
- 是否理解核心命题
- 是否理解关键因果
- 是否具备基础迁移
- 误解主要属于哪一类
- 下一步是推进、桥接还是重讲

## Final phase
topic 不会因为“lesson 讲完了”就关闭。
只有 synthesis、transfer、articulation 都完成并通过后，才能真正闭环。

## When to read the full rules
如果你正在生成真实 workspace 内的 lesson、diagnosis 或 final 文件，再读工作区里的 `AGENTS.md` 获取完整细则。
